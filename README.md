## 📌 Описание проекта

Верстка многостраничного/одностраничного макета [название источника: Figma/PSD] с акцентом на семантическую верстку, доступность (a11y), оптимизацию производительности и строгое следование методологии БЭМ.

Проект демонстрирует навыки работы с современным фронтенд-инструментарием и понимание Core Web Vitals.

## 🛠 Стек технологий

В проекте реализованы и использованы следующие подходы и инструменты:

| Технология                         | Реализация                                                                                                                                                                                                                  |
| :--------------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **HTML5**                          | Семантические теги (`<header>`, `<main>`, `<section>`, `<article>`, `<nav>`, `<footer>`). Использованы WAI-ARIA роли (`role`, `aria-label`, `aria-labelledby`) для обеспечения доступности.                                 |
| **CSS3 / SCSS**                    | Препроцессор SCSS. Применена строгая методология **БЭМ** (Block-Element-Modifier) для именования классов. Структура файлов организована по компонентному принципу.                                                          |
| **JavaScript (ES6+) / TypeScript** | Код написан на **TypeScript** с четкой типизацией. Использованы ES-модули (благодаря Vite).                                                                                                                                 |
| **Работа с изображениями**         | Реализована адаптивная графика: использован тег `<picture>` с атрибутом `srcset` для ретина-дисплеев (`@1x`, `@2x`). Все изображения сконвертированы в современный формат **WebP** с fallback на PNG/JPG.                   |
| **Оптимизация загрузки**           | Применены Resource Hints: `<link rel="preconnect">` для внешних источников шрифтов, `<link rel="preload">` для критичных шрифтов и изображений в первом экране, `<link rel="prefetch">` для фоновых изображений ниже сгиба. |
| **Метрики производительности**     | Проект собран с настройками минификации (`terser`), хешированием файлов для борьбы с кешированием и автоматическим разделением кода (code splitting) благодаря Vite. Целевой показатель Lighthouse ≥ 90.                    |
| **Сборщик**                        | **Vite** + **Bun**. Настроен быстрый сервер разработки и продакшен-сборка с оптимизацией ассетов.                                                                                                                           |

## 🚀 Установка и запуск

### Требования

- [Bun](https://bun.sh/) (рекомендуется) или Node.js (v18+).

### Команды

```bash
# 1. Клонировать репозиторий
git clone https://github.com/Alexey5689/testOnePage.git
cd [название папки]

# 2. Установить зависимости (через Bun)
bun install

# 3. Запустить режим разработки (горячая перезагрузка)
bun run dev

# 4. Собрать продакшен-версию (папка dist)
bun run build

# 5. Посмотреть собранную версию локально (опционально)
bun run preview
```
